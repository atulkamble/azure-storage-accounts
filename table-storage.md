## 🔹 What is a Storage Account Key?

A **Storage Account Key** is a **secret credential** generated by **Microsoft Azure** that provides **full access** to all services inside a Storage Account:

* Blob Storage
* File Storage
* Queue Storage
* Table Storage

📌 Think of it as a **root password** for the storage account.

---

## 🔑 Where Do You Get the Storage Account Key?

You get it from the **Azure Portal** under your **Storage Account**.

---

## 🪜 Step-by-Step (Azure Portal)

1. Login to **Azure Portal**
2. Open **Storage Accounts**
3. Select your **Storage Account**
4. Left menu → **Security + networking**
5. Click **Access keys**
6. Copy **Key1** or **Key2**

✅ Either key works
✅ Two keys are provided for **zero-downtime rotation**

---

## 📌 What You’ll See in Access Keys

| Field                | Usage                    |
| -------------------- | ------------------------ |
| Storage account name | `AZURE_STORAGE_ACCOUNT`  |
| Key1                 | Primary access key       |
| Key2                 | Secondary key (rotation) |

---

```bash
export AZURE_STORAGE_ACCOUNT=mystorageaccount
```

## 📌 What is Azure Table Storage?

Azure Table Storage is a **NoSQL key-value storage service** designed for:

* Massive scale
* Fast lookups
* Schema-less data

📦 Best for:

* User profiles
* Metadata
* Device telemetry
* Configuration data

---

## 🧱 Core Concepts (Must Know)

| Term         | Meaning                |
| ------------ | ---------------------- |
| Table        | Collection of entities |
| Entity       | One row                |
| Property     | Column                 |
| PartitionKey | Data grouping          |
| RowKey       | Unique ID              |

🔑 **PartitionKey + RowKey = Primary Key**

---

## ⚙️ Authentication (Basic)

```bash
export AZURE_STORAGE_ACCOUNT=mystorage
export AZURE_STORAGE_KEY=<key>
```

---

## 🧪 Azure CLI – BASIC Commands

### 1️⃣ Create Table

```bash
az storage table create \
  --name demoTable \
  --account-name $AZURE_STORAGE_ACCOUNT \
  --account-key $AZURE_STORAGE_KEY
```

---

### 2️⃣ Insert Entity

```bash
az storage entity insert \
  --table-name demoTable \
  --entity PartitionKey=Users RowKey=1 Name=Atul City=Pune \
  --account-name $AZURE_STORAGE_ACCOUNT \
  --account-key $AZURE_STORAGE_KEY
```

---

### 3️⃣ Query Entities

```bash
az storage entity query \
  --table-name demoTable \
  --account-name $AZURE_STORAGE_ACCOUNT \
  --account-key $AZURE_STORAGE_KEY
```

---

### 4️⃣ Update Entity

```bash
az storage entity merge \
  --table-name demoTable \
  --entity PartitionKey=Users RowKey=1 City=Mumbai \
  --account-name $AZURE_STORAGE_ACCOUNT \
  --account-key $AZURE_STORAGE_KEY
```

---

### 5️⃣ Delete Entity

```bash
az storage entity delete \
  --table-name demoTable \
  --partition-key Users \
  --row-key 1 \
  --account-name $AZURE_STORAGE_ACCOUNT \
  --account-key $AZURE_STORAGE_KEY
```

---

### 6️⃣ Delete Table

```bash
az storage table delete \
  --name demoTable \
  --account-name $AZURE_STORAGE_ACCOUNT \
  --account-key $AZURE_STORAGE_KEY
```

## ⚡ PowerShell – Basic

### Create Context

```powershell
$ctx = New-AzStorageContext `
  -StorageAccountName "mystorage" `
  -StorageAccountKey "<key>"
```

---

### Create Table

```powershell
New-AzStorageTable -Name "demoTable" -Context $ctx
```

---

### Insert Entity

```powershell
Add-AzTableRow `
  -Table "demoTable" `
  -PartitionKey "Users" `
  -RowKey "1" `
  -Property @{ Name="Atul"; City="Pune" } `
  -Context $ctx
```
